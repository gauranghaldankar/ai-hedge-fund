# FLW-005 — Add Kite Connect module-level singleton to `src/tools/api.py`

**Status:** Ready for dev
**Priority:** Must (blocks FLW-006)
**MoSCoW rationale:** FLW-006 cannot be implemented without the singleton; the singleton is the minimal,
zero-agent-change way to make Kite available inside `get_prices()`.
**Spec refs:** `docs/analysis/flow-kite-migration-pm-report.md` §3, §5

---

## Context

`api.py` already holds a module-level `_cache` singleton. We extend the same pattern to hold a Kite
client (`KiteConnect` instance) and instrument token lookup (`dict[str, int]`) so they are initialized
once per process and shared across all callers of `get_prices()`.

The singleton must be initialized lazily (on first yf-routed price request, not at module import time)
so that the module remains importable when `kiteconnect` is not installed or Kite credentials are absent.

This ticket covers only the singleton machinery. The actual price-fetch branch goes in FLW-006.

---

## User Stories

As a developer, I want Kite Connect initialized once per process when the first Indian-ticker price
request arrives, so that all subsequent agent calls in the same flow run reuse the same authenticated
client without paying an extra `instruments()` call per agent.

---

## Acceptance Criteria

**AC-0222** — Module-level variables `_kite_client` and `_kite_token_lookup` are defined in
`src/tools/api.py`, both initialized to `None`.

**AC-0223** — A private function `_init_kite()` in `src/tools/api.py` initializes `_kite_client`
and `_kite_token_lookup` (module globals) exactly once: if `_kite_client` is already set, the function
returns immediately without making any network call or re-importing `kiteconnect`.

**AC-0224** — `_init_kite()` is tolerant of all failure modes: if `kiteconnect` is not installed,
if `KITE_API_KEY` is absent, or if `build_token_lookup()` raises, it logs a debug-level message and
leaves both globals as `None`. It never raises an exception.

---

## Out of Scope

- Calling `_init_kite()` from anywhere (that is FLW-006).
- Changing any agent files.
- Changing `AgentState`.
- BSE/`.BO` support (deferred — Kite `instruments("NSE")` only).
- Thread-safety hardening (acknowledged risk, same as existing `_cache` singleton).

---

## Definition of Done

- AC-0222, AC-0223, AC-0224 each have a corresponding test in FLW-007.
- `poetry run pytest tests/tools/test_api_kite.py` passes.
- `get_prices()` behavior for non-.NS tickers is unchanged (verified by existing tests).
