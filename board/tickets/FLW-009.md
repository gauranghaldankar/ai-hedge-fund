# FLW-009 — Wire `FlowRunRepository` into `POST /hedge-fund/run`

**Status:** Ready for dev (depends on FLW-008)
**Priority:** Must
**MoSCoW rationale:** This is the persistence write path. Without it, no run data ever reaches the DB and the restore feature (FLW-011) has nothing to read.
**Depends on:** FLW-008 (flow_id field on request schema)
**Blocks:** FLW-011
**Spec refs:** `docs/requirements/flow-run-persistence.md` — AC-0234, AC-0235, AC-0236, AC-0237

---

## Problem Statement

`app/backend/routes/hedge_fund.py` streams analyst signals and portfolio decisions to the frontend but writes nothing to the database. `FlowRunRepository` (with `create_flow_run`, `update_flow_run`) and `HedgeFundFlowRun` (the DB model) exist but are never called. This ticket wires them into the existing `event_generator()` coroutine inside the `/run` handler.

## Implementation Notes

### 1. Add imports (top of file, after existing imports)

```python
from app.backend.repositories.flow_run_repository import FlowRunRepository
from app.backend.models.schemas import FlowRunStatus
```

### 2. Create the run record before graph execution starts

Inside `event_generator()`, before `run_task = asyncio.create_task(...)` (currently line 79), add:

```python
# Persist run to DB if a flow_id was provided
flow_run_id: int | None = None
if request_data.flow_id:
    run_repo = FlowRunRepository(db)
    flow_run = run_repo.create_flow_run(
        flow_id=request_data.flow_id,
        request_data=request_data.model_dump(exclude={"api_keys"})
    )
    run_repo.update_flow_run(run_id=flow_run.id, status=FlowRunStatus.IN_PROGRESS)
    flow_run_id = flow_run.id
```

Note: `api_keys` is excluded from `request_data` serialization — never persist credentials to the DB.

### 3. Update run to COMPLETE when the CompleteEvent is built

After `final_data = CompleteEvent(...)` is constructed (currently lines 130–136) and before `yield final_data.to_sse()`, add:

```python
if flow_run_id:
    run_repo = FlowRunRepository(db)
    run_repo.update_flow_run(
        run_id=flow_run_id,
        status=FlowRunStatus.COMPLETE,
        results=final_data.data
    )
```

`final_data.data` is already a plain dict (`{"decisions": ..., "analyst_signals": ..., "current_prices": ...}`). It is directly storable in the `results` JSON column.

### 4. Update run to ERROR on failure

In the `except asyncio.CancelledError` branch and in the block after `if not result or not result.get("messages"):`, add an error update:

```python
if flow_run_id:
    run_repo = FlowRunRepository(db)
    run_repo.update_flow_run(
        run_id=flow_run_id,
        status=FlowRunStatus.ERROR,
        error_message="Run was cancelled or produced no output"
    )
```

For the general exception path, capture the exception message:

```python
except Exception as exc:
    if flow_run_id:
        run_repo = FlowRunRepository(db)
        run_repo.update_flow_run(
            run_id=flow_run_id,
            status=FlowRunStatus.ERROR,
            error_message=str(exc)
        )
    raise
```

### 5. The `db` session

`db: Session` is already injected into the outer `run()` function via `Depends(get_db)` (line 26). The `event_generator()` closure captures it. This is an existing pattern — the `event_generator()` already uses `request_data` and `portfolio` from the outer scope. The same closure capture applies to `db`.

Important: `db` is a synchronous SQLAlchemy session. The `FlowRunRepository` calls are synchronous. Do not wrap them in `asyncio.run_in_executor` — they are short, non-blocking DB writes that are acceptable to run synchronously inside an async generator for this use case. If this causes latency in a multi-user scenario that is a future concern, not in scope here.

### 6. No-op when flow_id is absent

Every code path above is gated on `if request_data.flow_id:` or `if flow_run_id:`. When `flow_id` is `None`, the function behaves exactly as today with zero additional overhead.

## Acceptance Criteria

**AC-0234** — After `POST /hedge-fund/run` is called with a valid `flow_id`, a row exists in `hedge_fund_flow_runs` with `status=IN_PROGRESS` and a non-null `started_at`.

**AC-0235** — After the run completes successfully, the row has `status=COMPLETE`, non-null `completed_at`, and `results` containing the `decisions`, `analyst_signals`, and `current_prices` keys.

**AC-0236** — After a run fails (exception or cancelled), the row has `status=ERROR` and a non-empty `error_message`.

**AC-0237** — A request without `flow_id` produces no DB writes and the SSE stream behaves identically to the pre-change behavior.
