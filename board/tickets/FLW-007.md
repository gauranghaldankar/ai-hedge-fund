## Context

A new test file `tests/tools/test_api_kite.py` covers the Kite-in-api.py surface. All Kite network
calls are mocked using `unittest.mock.patch` and `MagicMock`, following the same pattern as
`src/screener/tests/test_kite_api.py`.

The existing `_cache` singleton must be reset between tests (or patched) to prevent cross-test
contamination. The `_kite_client` and `_kite_token_lookup` globals added in FLW-005 must also be
reset between tests (patch them to `None` at the start of each test that exercises init logic).

---

## User Stories

As an engineer, I want a test suite that proves the Kite branch in `api.py` works correctly
(and that the fallback to yfinance works when Kite is absent), so that regressions are caught
automatically.

---

## Acceptance Criteria

**AC-0229** â€” `_init_kite()` is called exactly once across two consecutive calls to `get_prices()`
for an `.NS` ticker within the same process: the second call reuses the cached `_kite_client` without
re-importing kiteconnect or calling `kite.instruments()` again. (Tests AC-0223.)

**AC-0230** â€” When `kiteconnect` is not installed (simulated by `patch`-ing the import to raise
`ImportError`), `_init_kite()` leaves `_kite_client` as `None` and does not raise. (Tests AC-0224.)

**AC-0231** â€” When a mock Kite client returns valid candles for `"SGFIN"`, `get_prices("SGFIN.NS", ...)`
returns a non-empty `list[Price]` with `open`/`close`/`high`/`low`/`volume` populated, and does NOT
call `yf_api.get_prices`. (Tests AC-0225, AC-0226.)

**AC-0232** â€” When the mock Kite client returns `None` (symbol not found), `get_prices("SGFIN.NS", ...)`
falls through and calls `yf_api.get_prices` exactly once. (Tests AC-0227.)

**AC-0233** â€” `get_prices("INFY.BO", ...)` and `get_prices("AAPL", ...)` do not call `_init_kite()` or
any Kite function. (Tests AC-0228 â€” `.BO` falls through to yfinance; US ticker goes to financialdatasets.)
