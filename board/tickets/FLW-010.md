# FLW-010 — Add `getLatestFlowRun` to `flow-service.ts` and define `FlowRunResponse` type

**Status:** Ready for dev
**Priority:** Must
**MoSCoW rationale:** The frontend needs a typed API call to fetch the last run for a given flow. Without it, `flow-context.tsx` (FLW-011) cannot call the backend and the restore path does not exist.
**Depends on:** Nothing (the backend endpoint `GET /flows/{flow_id}/runs/latest` already exists)
**Blocks:** FLW-011

## Problem Statement

`app/frontend/src/services/flow-service.ts` contains methods for flow CRUD (`getFlows`, `getFlow`, `createFlow`, `updateFlow`, `deleteFlow`, `duplicateFlow`) but has no method to read run history. The backend route `GET /flows/{flow_id}/runs/latest` (in `flow_runs.py`) already exists and is fully functional — it just has no frontend caller.

## Implementation Notes

### 1. Add the `FlowRunResponse` interface

Add at the top of the file, after the existing `UpdateFlowRequest` interface (after line 25):

```typescript
export interface FlowRunResponse {
  id: number;
  flow_id: number;
  status: 'IDLE' | 'IN_PROGRESS' | 'COMPLETE' | 'ERROR';
  run_number: number;
  created_at: string;
  updated_at: string | null;
  started_at: string | null;
  completed_at: string | null;
  request_data: Record<string, unknown> | null;
  results: Record<string, unknown> | null;
  error_message: string | null;
}
```

`results` is typed as `Record<string, unknown>` rather than a specific shape because the output node already uses `OutputNodeData` from the node context for its rendering — `flow-service.ts` does not need to know the internal structure. The caller in `flow-context.tsx` passes it straight through to `nodeContext.setOutputNodeData()`.

### 2. Add `getLatestFlowRun` to the `flowService` object

Add after the `createDefaultFlow` method (after line 107):

```typescript
// Get the most recent run for a flow (returns null if no runs exist)
async getLatestFlowRun(flowId: number): Promise<FlowRunResponse | null> {
  const response = await fetch(`${API_BASE_URL}/flows/${flowId}/runs/latest`);
  if (!response.ok) {
    throw new Error(`Failed to fetch latest flow run: ${response.status}`);
  }
  return response.json(); // backend returns null body when no run exists
},
```

The backend returns `null` (HTTP 200 with a null JSON body) when no run exists for the flow. `response.json()` will parse that correctly to `null` in JavaScript.

No new imports are needed — `fetch` and `API_BASE_URL` are already in scope.
