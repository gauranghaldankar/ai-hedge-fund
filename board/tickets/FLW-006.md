# FLW-006 — Kite price branch inside `api.py::get_prices()` for .NS tickers

**Status:** Ready for dev (depends on FLW-005)
**Priority:** Must
**MoSCoW rationale:** Without this, Indian ticker price data in Flows continues to come from yfinance
(unreliable intraday, slow, subject to 401 noise). Kite provides genuine exchange-level daily OHLCV.
**Spec refs:** `docs/analysis/flow-kite-migration-pm-report.md` §3, §4, §6, §7

---

## Context

`api.py::get_prices()` currently routes `.NS`/`.BO`/`.BSE` tickers to `yf_api.get_prices()` via
`_is_yf_ticker()`. We insert a Kite-primary path before the yfinance path for `.NS` tickers only.

The screener's `_fetch_ticker_data()` is the reference implementation (see
`src/screener/run_screener.py` lines 117-153).

Key differences from screener:
- The screener passes `kite_client` and `kite_token_lookup` as explicit function params.
- In `api.py`, these are the module-level globals from FLW-005 — no signature change to `get_prices()`.
- The screener receives a pre-built DataFrame; `get_prices()` must convert the Kite DataFrame
  (columns: open/high/low/close/volume, DatetimeIndex) into `list[Price]` — the same type it already
  returns for every other path. The conversion maps each row to a `Price` object using the row's
  index as the `time` field.

The `.BO` and `.BSE` paths continue to route directly to yfinance unchanged (Kite's `instruments("NSE")`
does not cover BSE). Only `.NS` tickers enter the Kite branch.

---

## User Stories

As a trader running a flow on `SGFIN.NS`, I want price data to come from Kite Connect (genuine NSE
exchange data) rather than yfinance, so that technical analysis indicators are computed on accurate
OHLCV history.

As a system operator, I want the Flows pipeline to continue working without any change if Kite
credentials are missing or Kite returns no data for a symbol, so that the app degrades gracefully.

---

## Acceptance Criteria

**AC-0225** — Inside `get_prices()`, when `_is_yf_ticker(ticker)` is True AND the ticker ends with
`.NS`: `_init_kite()` is called, and if `_kite_client` and `_kite_token_lookup` are both non-None,
`get_price_data_kite()` is attempted with the bare symbol
(`ticker.removesuffix(".NS")`) before any yfinance call.

**AC-0226** — If `get_price_data_kite()` returns a non-empty DataFrame, the function converts it to
`list[Price]` (one `Price` per row, `time` formatted as `"%Y-%m-%dT00:00:00"`, lowercase column names
`open`/`close`/`high`/`low`/`volume`), caches the result under the key `kite_{ticker}_{start}_{end}`,
and returns the list. yfinance is NOT called.

**AC-0227** — If `get_price_data_kite()` returns `None` or an empty DataFrame, OR if `_kite_client`
is `None` (Kite not configured or init failed), the function falls through to the existing
`yf_api.get_prices()` call unchanged. No error is raised.

**AC-0228** — `.BO` and `.BSE` tickers bypass the new Kite branch entirely and continue to route to
`yf_api.get_prices()` as before. No behavior change for non-`.NS` yf tickers or for US tickers.

---

## Out of Scope

- `get_price_data()` does not need to change; it calls `get_prices()` which already handles the routing.
- `prices_to_df()` does not change.
- No changes to `yf_api.py`.
- No changes to any agent file (`technicals.py`, `risk_manager.py`, `nassim_taleb.py`,
  `stanley_druckenmiller.py`).
- No changes to the backtesting engine (it uses US tickers; `_is_yf_ticker()` returns False).
- BSE exchange Kite support is deferred.
- Cache invalidation / TTL is a separate concern (FLW-004).
